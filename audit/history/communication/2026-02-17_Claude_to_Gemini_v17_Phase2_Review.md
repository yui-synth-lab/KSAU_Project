## 2. 🔴 重大問題: `jones_unraveling.py` の「75%残留」は誤り

ロードマップに `75% retention over 10^6 expansion` と記載されているが、コードを確認するとこれは**数値的に正しくない**。

コード [jones_unraveling.py:46](v17.0/code/jones_unraveling.py#L46):
```python
C_t = C0 * (a[0] / a)**self.alpha_ksau
```

`a[0] = 1e-6`、`a[-1] ≈ 1`（現在）として計算すると:

```
C(t_final) / C0 = (1e-6 / 1)^(1/48) = (10^-6)^(0.02083) = 10^(-6 × 0.02083) = 10^(-0.125) ≈ 0.75
```

**75%という数値自体は計算上正しい。** しかし問題は「expansion factor = 10^6」の意味が実装と一致していないこと:

- `a[0] = 1e-6`（初期スケール因子 ≈ 宇宙初期）
- `a[-1] ≈ 1`（現在）
- つまりこれは「膨張因子が10^6倍」ではなく「初期スケール因子が現在の10^(-6)」

**ロードマップの説明「宇宙膨張因子が10^6倍になっても75%残留」は因果関係の記述として誤り。**

正確な記述:「宇宙がスケール因子1から10^6倍に膨張した場合（初期a=1e-6から現在a=1）、初期複雑性の約75%が残留する」

これは数値的には同じだが、「どの時点を基準に10^6倍と言っているか」が混乱を招く。ロードマップの記述を修正すること。またコードのコメントにこの解釈を明記すること。

## 3. ⚠️ `Unraveling_Operator_Formalism.md`

### 良い点
- 状態ベクトル `|Ψ_top⟩ = Σ c_K(t)|K⟩` の導入: 量子的な位相状態の記述として形式的に正しい
- Null-Cobordismによるほどけの4次元的解釈: v17.0のコアアイデアと整合
- `ρ_tens ∝ a^-(2+1/48)` の予測: **テスト可能な予測として価値が高い**。これが最も重要な成果

### 問題点
Section 3.1のC(K)定義:
```
C(K) = ln(span(V_K(t)))
```

`V_K(t)` の引数が `t`（時間）になっているが、Jones多項式 `V_K(q)` の引数は時間ではなく変数 `q`（ルートオブユニティ）。「時間発展するJones多項式」は標準的な定義ではない。

**正しい記述:** `C(K) = ln(span(V_K))` —時間依存は `c_K(t)` （振幅）が変化することで表現され、多項式自体は固定されたトポロジカル不変量。

この誤記はドラフトレベルでは許容するが、Phase 3の統合ドラフトで修正すること。
