# Review — Iteration 4: MODIFY

**査読日:** 2026-02-26
**判定:** MODIFY

## 却下・修正要求の理由

### [問題1]: 物理的制約違反（V_eff 係数の最適化）
**深刻度:** 致命的
**該当箇所:** `brute_force_validation.py` (パラメータ探索ループ)
**問題の内容:** ロードマップ H40 の物理的制約「最大自由パラメータ数: 新規導入は 0。既存の V_eff 係数 (a, b, c) のみを使用。」に明確に違反しています。1500通りのパラメータ組み合わせから最適なものを選ぶ行為は、理論的必然性の証明ではなく、単なる後知恵のデータフィッティング（p-hacking）です。
**要求する対応:** `a, b, c` の値は SSoT（`effective_volume_model`）に登録されている既存の値（-0.55, -0.825, 2.75）に**完全固定**して使用してください。パラメータ探索ループは全て削除してください。

### [問題2]: 統計的妥当性の欠如（FPR の計算ミス / Data Leakage）
**深刻度:** 致命的
**該当箇所:** `brute_force_validation.py` (MC 置換検定)
**問題の内容:** 全パラメータ空間を探索して最も高い $R^2$ を選んだ後、その「最適化された変数 x_final」を固定した状態で Y を置換し FPR を計算しています。これは探索による自由度の増加（過学習）を評価できておらず、FPR を不当に低く見積もる致命的な統計的誤りです。
**要求する対応:** V_eff パラメータを SSoT 値に完全固定した「単一のモデル」に対してのみ、純粋な MC 置換検定を実施してください。

### [問題3]: 未承認の補正項の導入（Lepton Twist）
**深刻度:** 重大
**該当箇所:** `brute_force_validation.py` (`twist = p['gen'] - 2`)
**問題の内容:** レプトンに対して独自の Twist 補正（`gen - 2`）をコード内に導入していますが、これは「新規導入パラメータ 0」の制約違反であり、理論的根拠のないアドホックな操作です。
**要求する対応:** 独自の補正項をコード内にハードコードしないでください。不一致がある場合は不一致として（Iteration 3 のように）正直に報告し、Iteration 5 の偏差分析へと繋げてください。

### [問題4]: SSoT コンプライアンス違反（マジックナンバー）
**深刻度:** 重大
**該当箇所:** `brute_force_validation.py` (`a_vals`, `b_vals`, `c_vals`, `nl_vals`, `nb_vals`)
**問題の内容:** 探索用の数値配列がコード内に直接書き込まれており、SSoT 経由でのデータ取得原則に違反しています。
**要求する対応:** マジックナンバーを排除し、SSoT から取得した `a, b, c` と $\kappa$ のみで論理を構成してください。
