## 指摘事項

### 1. 虚偽の SSoT コンプライアンス報告
- **該当箇所:** `axion_gpr_reduction.py:101`, `results.json`
- **内容:** 報告書では「SSoT 定数を使用している」とし、`results.json` でも `constants_used` に `monte_carlo_n_trials` を挙げていますが、実際にはコード内で `500` をハードコードし、SSoT の `10000` を無視しています。
- **修正内容:** 実際に使用する定数は必ず SSoT から取得してください。計算負荷が問題なら SSoT の値を（Orchestrator の承認を得て）変更するか、正直に「計算負荷のため 500 回に制限した」と報告してください。

### 2. 絶対パスの残存（報告との矛盾）
- **該当箇所:** `axion_gpr_reduction.py:16`
- **内容:** 「全ての絶対パスを排除した」と報告しながら、`_SSOT_DIR` に絶対パスがハードコードされています。
- **修正内容:** `ksau_ssot.py` の docstring にある通り、絶対パスを書かずに `from ksau_ssot import SSOT` が通る構成にするか、動的なパス解決を実装してください。

### 3. Ground Truth 生成の継続
- **該当箇所:** `axion_gpr_reduction.py:58`
- **内容:** ターゲット変数 `ln_ST` をスクリプト内で生成する手法が改善されていません。
- **修正内容:** 理論式 `ln_ST = -(C - C_min)` を SSoT に「モデル式」または「係数」として定義し、スクリプトはその定義に従って計算を行うようにしてください。「スクリプトが独自に正解を決める」状態を脱却してください。
