## 1. 必達タスク（v32.0 技術的負債の解消）

### Task A: ERR_THRESH 循環閾値の解消（優先度: HIGH、放置不可）

**問題の詳細（v32.0 go.md §3 [負債 #1] より）:**

```python
err_7       = abs(ratio_r4_rs - 7.0) / 7.0    # 観測誤差から閾値を設定
ERR_THRESH  = err_7                              # ← 循環：観測値自身が有意性の閾値
```

`N^{1/4}/r_s ≈ 7` の誤差 `err_7 = 0.2044%` を MC 検定の合格閾値に用いることは、「この観測値と同程度の精度の一致」を検出するよう検定を設計することと等しい。結果として MC p 値が過小評価方向にバイアスする。

**v33.0 での対応方針:**

- **(a) 計測誤差の公式不確かさを使用**: $r_s$ の Planck 2018 不確かさ（$\sigma_{r_s}$）から独立な相対誤差閾値を設定する。
- **(b) スケール独立な相対誤差基準**: 観測データに依存しない先験的な許容誤差（例: 1%, 5%, 10%）を複数設定し、結論の閾値依存性を明示する。
- **(c) SSoT への格納**: 選択した閾値を `v6.0/data/cosmological_constants.json` または `physical_constants.json` に格納し、コードからハードコードを排除する。

**成功基準**: 独立な根拠に基づく閾値設定完了、新閾値での MC 再実行、結果の比較（バイアス方向の定量化）。

### Task B: MC 乱数シード安定性検証（優先度: MEDIUM）

**問題の詳細（v32.0 go.md §3 [負債 #2] より）:**

`random.seed(42)` 固定で再現性は確保されているが、特定のシードが結果に有利に働いていないかの確認が未実施。

**v33.0 での対応方針:**

- 複数シード（例: 0, 1, 7, 42, 100, 314）での MC 再実行。
- 各シードでの p 値分布を記録。
- 結果の安定性（標準偏差、最大値-最小値差）を定量化。

**成功基準**: 複数シードで p 値が一貫して `p > 0.0024`（Bonferroni 補正後）であることを確認（または例外の記録）。

## 2. v33.0 新規目標 (Core Objectives)

### Section A: 非標準 WZW の系統的文献探索（最重要）

v31.0 go.md §3 [負債 #3] および v32.0 go.md §3 [負債 #3] より。現フェーズ最後の残存代数的経路。

**探索対象:**

1. **Curved background の WZW**
   - $\pi$ は経路積分の位相因子として出現しうる（$e^{i\pi \cdot k}$ 型）
   - Witten (1984) の原論文および後続文献の再精査
   - 「Curved background WZW model」「WZW on group manifold with curvature」キーワードで arXiv 探索

2. **Coset WZW 構成（$G/H$ 型）**
   - $SU(N)/H$ 型のコセット構成で `7` が自然に出現するかを調査
   - 特に $SU(24)/SU(23)$、$SU(7)/...$、$G_2/SU(3)$ 等の候補
   - GKO（Goddard-Kent-Olive）構成との照合

3. **非コンパクト WZW**
   - 非コンパクトゲージ群（$SL(2,\mathbb{R})$、$SU(1,1)$等）の WZW 理論
   - AdS₃/CFT₂ 文脈での非コンパクト WZW
   - $7\pi/k$ 型の係数出現可能性の評価

**判定基準:**
- 「可能性あり（具体的な計算で確認すべき経路が見つかった）」
- 「不可能と確定（対象の数学的構造が $7\pi/k$ を生成しない理由が明確）」
- 「未解決（文献なし / 計算が困難で現段階では判断不可）」

**成功基準**: 3ケース各々について上記いずれかの結論を証拠に基づいて宣言。「調査中」での持ち越し禁止。
